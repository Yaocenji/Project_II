# 脚本 ProjectII.Item.Hotbar
继承：MonoBehaviour

快捷栏脚本，与背包并列独立，玩家持有一个。
负责**统一监听输入**并将交互**转发**给当前装备的物品。
快捷栏是整个物品使用系统的核心枢纽。

## 成员 物品列表

类型：Item.Base 数组。
存储快捷栏中的物品实例。
如果该位置为空值，那么相当于这个格子没东西。

快捷栏与背包是独立的，物品在两者之间互相移动。
快捷栏不支持堆叠，每格只存一个物品实例。
非当前装备格的物品处于 SetActive(false) 状态。

## 方法 放入物品

参数：物品（Item.Base）、目标格子序号。
如果该位置没有物品，那么放入物品；
如果该位置有物品，那么拒绝放入。
放入成功后：如果目标格是当前装备格，SetActive(true)；否则 SetActive(false)。

## 方法 拿出物品

参数：目标格子序号。
如果该位置有物品就拿出，否则拒绝拿出。
拿出时将物品 SetActive(true)（调用方自行决定后续显隐）。

## 成员 当前装备的物品序号

必须是 0 ~ 物品列表长度-1 之间；
如果该格子为空，就相当于是空手。

## 成员 InputAction 引用

从 InputManager 处获取当前玩家的 InputAction 引用。
快捷栏统一监听以下输入：mainAttack、secondaryAttack、reload。

## 输入状态缓存

维护以下布尔状态，用于在 Update 中检测持续按下（Hold）：
- mainAttackIsHolding
- secondaryAttackIsHolding

## Awake 初始化

从 InputManager 获取 InputAction 引用，并注册以下输入回调：
- mainAttack.started → 触发主要交互 Press
- mainAttack.canceled → 触发主要交互 Release
- secondaryAttack.started → 触发次要交互 Press
- secondaryAttack.canceled → 触发次要交互 Release
- reload.started → 触发装填 Press

## Update 轮询

每帧检测：
- 如果 mainAttack 处于持续按下状态，调用当前装备物品的 MainInteractHold
- 如果 secondaryAttack 处于持续按下状态，调用当前装备物品的 SecondaryInteractHold

## 方法 触发物品的主要交互 Press

由 InputAction 的 mainAttack.started 触发。
如果当前装备的物品为空，则什么都不做；
否则，调用当前装备物品的 MainInteractPress。

## 方法 触发物品的主要交互 Hold

由 Update 中检测 mainAttack 持续按下时触发。
如果当前装备的物品为空，则什么都不做；
否则，调用当前装备物品的 MainInteractHold。

## 方法 触发物品的主要交互 Release

由 InputAction 的 mainAttack.canceled 触发。
如果当前装备的物品为空，则什么都不做；
否则，调用当前装备物品的 MainInteractRelease。

## 方法 触发物品的次要交互 Press

由 InputAction 的 secondaryAttack.started 触发。
如果当前装备的物品为空，则什么都不做；
否则，调用当前装备物品的 SecondaryInteractPress。

## 方法 触发物品的次要交互 Hold

由 Update 中检测 secondaryAttack 持续按下时触发。
如果当前装备的物品为空，则什么都不做；
否则，调用当前装备物品的 SecondaryInteractHold。

## 方法 触发物品的次要交互 Release

由 InputAction 的 secondaryAttack.canceled 触发。
如果当前装备的物品为空，则什么都不做；
否则，调用当前装备物品的 SecondaryInteractRelease。

## 方法 触发物品的装填

由 InputAction 的 reload.started 触发。
如果当前装备的物品为空，则什么都不做；
否则，调用当前装备物品的 ReloadPress。

## OnDestroy 清理

取消注册所有输入回调，防止内存泄漏。

## 成员 背包引用

类型：Backpack。
在 Awake 中通过 GetComponent 获取同 GameObject 上的 Backpack 引用。
用于物品转移操作。

## 方法 从快捷栏转移到背包 TransferToBackpack

参数：快捷栏格子序号（int hotbarSlot）、背包格子序号（int backpackSlot）。
返回 bool，表示是否转移成功。

同步函数，调用即完成，不存在"物品悬空"的中间态：
1. 校验：快捷栏该格子是否有物品，背包该格子是否能放入；
2. 如果转移的是当前装备格的物品，先调用 OnUnequip；
3. 从快捷栏取出物品（TakeItem）；
4. 尝试放入背包（Backpack.PutItem）——背包 PutItem 内部会将物品 SetActive(false)；
5. 如果放入失败，将物品放回快捷栏原位（回滚），并恢复正确的 SetActive 状态。

## 方法 从背包转移到快捷栏 TransferFromBackpack

参数：背包格子序号（int backpackSlot）、快捷栏格子序号（int hotbarSlot）。
返回 bool，表示是否转移成功。

同步函数，调用即完成：
1. 校验：背包该格子是否有物品，快捷栏该格子是否为空；
2. 从背包取出物品（Backpack.TakeItem）——背包 TakeItem 内部会将物品 SetActive(true)；
3. 尝试放入快捷栏（PutItem）——PutItem 内部根据是否为当前装备格设置 SetActive；
4. 如果放入失败，将物品放回背包原位（回滚）；
5. 如果放入成功且该格是当前装备格，调用 OnEquip。

## 方法 快捷栏内交换 SwapHotbarSlots

参数：格子序号 A（int slotA）、格子序号 B（int slotB）。
返回 bool，表示是否交换成功。

同步函数，调用即完成：
1. 校验：两个序号合法且不相同；
2. 直接交换两个格子的物品引用；
3. 如果其中一个格子是当前装备格，需要正确触发 OnUnequip / OnEquip；
4. 交换后更新两个格子物品的 SetActive 状态（当前装备格 true，其余 false）。

## 方法 丢弃快捷栏物品 DropItemFromHotbar

参数：快捷栏格子序号（int hotbarSlot）、丢弃位置（Vector2 dropPosition）。
返回 bool，表示是否丢弃成功。

同步函数，调用即完成：
1. 校验：快捷栏该格子是否有物品，物品的 worldItemPrefab 是否存在；
2. 如果丢弃的是当前装备格的物品，先调用 OnUnequip；
3. 从快捷栏取出物品（TakeItem）；
4. 根据物品的 worldItemPrefab 在 dropPosition 处实例化掉落物；
5. 销毁手持物 GameObject（Destroy）；
6. 返回 true。

## 方法 丢弃背包物品 DropItemFromBackpack

参数：背包格子序号（int backpackSlot）、丢弃位置（Vector2 dropPosition）。
返回 bool，表示是否丢弃成功。

同步函数，调用即完成：
1. 校验：背包该格子是否有物品，零号物品的 worldItemPrefab 是否存在；
2. 从背包取出物品（Backpack.TakeItem）；
3. 根据物品的 worldItemPrefab 在 dropPosition 处实例化掉落物；
4. 销毁手持物 GameObject（Destroy）；
5. 返回 true。

## 设计说明：物品转移

物品转移是一个**同步函数**，调用即完成。
UI 层的拖拽只是视觉表现——拖拽过程中，物品在数据层仍然待在原来的格子里，没有任何状态变化。
真正的转移发生在"放下"的那一瞬间：校验 → 取出 → 放入，要么全部成功，要么全部不做（回滚）。
逻辑层永远不会出现"物品既不在背包也不在快捷栏"的悬空状态。

转移逻辑统一由 Hotbar 管理，因为 Hotbar 本身就是物品操作的核心枢纽，
它天然持有快捷栏数据，再加一个 Backpack 引用即可完成跨容器转移，无需额外新建管理器。
以后如果 UI 层需要触发转移（如拖拽、Shift+点击快速转移），只需调用 Hotbar 上的对应方法即可。

## 设计说明：输入转发

快捷栏的输入监听逻辑与原 WeaponBase 中的输入监听逻辑完全一致（Press/Hold/Release 三态模型），
只是将监听位置从武器自身提升到了快捷栏层级，实现了"物品不直接监听输入"的设计原则。
这样做的好处是：所有物品（武器、道具、工具等）共享同一套输入转发机制，无需各自重复实现输入监听。

## 设计说明：物品显隐（SetActive）管理

快捷栏中的物品 GameObject 显隐规则：
- **当前装备格**的物品：SetActive(true)，玩家可以看到并使用；
- **非当前装备格**的物品：SetActive(false)，隐藏不可见。

这一规则在以下操作中统一维护：
- PutItem：放入时根据是否为当前装备格决定显隐；
- TakeItem：取出时统一 SetActive(true)，后续由调用方处理；
- SwitchSlot：切走的物品 SetActive(false)，切入的物品 SetActive(true)；
- TransferToBackpack / TransferFromBackpack：由各自内部和背包的 PutItem/TakeItem 协同管理；
- SwapHotbarSlots：交换后根据当前装备格重新设置显隐；
- DropItem：取出后直接销毁，无需额外处理显隐。

背包中的物品始终处于 SetActive(false) 状态，由 Backpack 自身的 PutItem/TakeItem 管理。

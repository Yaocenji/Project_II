# 脚本 ProjectII.Weapon.WeaponBase
继承：ProjectII.Item.Base（即间接继承 MonoBehaviour）

所有武器的基类。
武器是物品的一种，继承自物品基类 Item.Base。
武器**不直接监听输入**，所有输入由快捷栏（Hotbar）统一监听后转发。

## 武器特有成员

- spreadAngle：武器散布角度（单侧、角度）
- range：武器射程（世界空间）
- canFire：能否开火标志位（由派生类根据具体规则设置）
- bulletPool：子弹对象池（ObjectPool<GameObject>，子类可重写初始化）

## 重写 Item.Base 的交互方法

WeaponBase 将 Item.Base 的 7 个虚方法重新声明为**纯虚方法（abstract）**，
强制所有具体武器子类必须实现：

### 纯虚方法 MainInteractPress（重写自 Item.Base）
主要攻击被按下的瞬间调用一次。

### 纯虚方法 MainInteractHold（重写自 Item.Base）
主要攻击按下的持续时间内，每个 Update 都被调用。

### 纯虚方法 MainInteractRelease（重写自 Item.Base）
主要攻击被抬起的瞬间调用一次。

### 纯虚方法 SecondaryInteractPress（重写自 Item.Base）
次要攻击被按下的瞬间调用一次。

### 纯虚方法 SecondaryInteractHold（重写自 Item.Base）
次要攻击按下的持续时间内，每个 Update 都被调用。

### 纯虚方法 SecondaryInteractRelease（重写自 Item.Base）
次要攻击被抬起的瞬间调用一次。

### 纯虚方法 ReloadPress（重写自 Item.Base）
重新装填按下时的回调。

## 设计说明

与旧版 WeaponBase 的区别：
1. **继承关系**：旧版直接继承 MonoBehaviour → 新版继承 Item.Base
2. **输入监听**：旧版自己持有 InputAction 引用并注册回调 → 新版不再监听输入，由 Hotbar 转发
3. **虚方法名称**：旧版 MainAttackPress 等 → 新版统一为 MainInteractPress 等（与 Item.Base 一致）
4. **子类影响**：具体武器子类（如 Revolver）只需将方法名从 MainAttackPress 改为 MainInteractPress，逻辑无需任何改动

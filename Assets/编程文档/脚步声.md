
# 脚本 ProjectII.Character.FootstepAudio
继承：MonoBehaviour
挂载位置：玩家角色 GameObject（或其子物体，与 StudioEventEmitter 同级）

脚步声音频控制器。
负责根据角色移动状态触发脚步声播放，独立于 CharacterController，遵循单一职责原则。

## 设计动机

原先脚步声的触发逻辑（距离累加、Play 调用、Emitter 引用）全部写在 CharacterController 中，
导致角色控制器承担了不属于它的音频职责，且 FMOD_FootStepPlayer_Parameter_Trigger 需要穿过
GameSceneManager → CharacterController → footstepSFX_Emitter 才能设置参数，耦合过深。

独立后：
- CharacterController 只管移动和旋转
- FootstepAudio 只管脚步声的触发和参数
- FMOD_FootStepPlayer_Parameter_Trigger 直接找 FootstepAudio 设置参数

## 公有数据

### StudioEventEmitter footstepEmitter
脚步声 FMOD 发射器引用。
在 Inspector 中赋值，指向同 GameObject（或子物体）上的 StudioEventEmitter 组件。

### float footstepDistanceInterval
触发一次脚步声所需的移动距离阈值，默认 0.75。
可在 Inspector 中调整。

## 私有数据

### Rigidbody2D rb
玩家刚体引用，Awake 中通过 GetComponentInParent 获取。
用于读取当前移动速度（rb.velocity.magnitude）。

### CharacterController characterController
玩家角色控制器引用，Awake 中通过 GameSceneManager 获取。
用于读取当前速度状态（CurrentSpeedState），以便在静止时不累加距离。

### float footstepDistance
当前已累加的移动距离，达到 footstepDistanceInterval 时触发播放并归零。

## 方法 Awake
- 通过 GetComponentInParent 获取 Rigidbody2D
- 通过 GameSceneManager.Instance.CurrentPlayerCharacter 获取 CharacterController

## 方法 FixedUpdate
核心逻辑，每物理帧执行：
1. 如果 CharacterController.CurrentSpeedState == Idle，不累加距离，直接返回
2. footstepDistance += Time.fixedDeltaTime × rb.velocity.magnitude
3. 如果 footstepDistance >= footstepDistanceInterval：
   - footstepDistance = 0
   - footstepEmitter.Play()

## 方法 SetParameter(string name, float value)
公有方法，供 FMOD_FootStepPlayer_Parameter_Trigger 调用，设置 Emitter 的 FMOD 参数。
内部直接调用 footstepEmitter.SetParameter(name, value)。

---

# 对 CharacterController 的修改

从 CharacterController 中移除以下内容：
- 字段 footstepSFX_Emitter
- 字段 footStepDistance
- 字段 footStepDistanceInterval
- FixedUpdate 中的脚步声距离累加和播放逻辑

CharacterController 不再持有任何音频相关引用。

---

# 对 FMOD_FootStepPlayer_Parameter_Trigger 的修改

原先的参数设置路径：
```
GameSceneManager.Instance.CurrentPlayerCharacter.footstepSFX_Emitter.SetParameter(...)
```

修改后的参数设置路径：
```
// 1. 通过 GameSceneManager 确认进入触发器的是玩家
//    （OnTriggerEnter2D 中 other.CompareTag("Player") 已经判断了）
// 2. 通过 GameSceneManager 拿到玩家角色，再从玩家角色上获取 FootstepAudio 组件
var playerCharacter = GameSceneManager.Instance.CurrentPlayerCharacter;
FootstepAudio footstepAudio = playerCharacter.GetComponentInChildren<FootstepAudio>();
footstepAudio.SetParameter("GroundMaterial", (float)groundMaterial);
```

说明：
- OnTriggerEnter2D 中已有 CompareTag("Player") 判断，确保只对玩家生效
- 仍然需要通过 GameSceneManager 拿到 CharacterController，因为 Collider2D 的 other 是物理碰撞体，
  不一定直接挂载 FootstepAudio；通过 GameSceneManager 拿到的是确定的玩家根对象，
  再 GetComponentInChildren 查找 FootstepAudio 更可靠
- 如果未来 NPC 也需要脚步声，NPC 也挂载 FootstepAudio，
  触发器可以改为直接从 other.GetComponentInParent<FootstepAudio>() 获取，不走 GameSceneManager

---

# 对 GameSceneManager 的修改

不需要修改。
GameSceneManager 仍然只注册 CharacterController 和 Mouse，不需要额外注册 FootstepAudio。
FootstepAudio 作为玩家角色的子组件，通过 GetComponentInChildren 即可获取。

---

# 使用方法

## 1. 搭建 FootstepAudio

1. 在玩家角色 Prefab 下创建一个子 GameObject，命名为 `FootstepAudio`（或其他有意义的名称）；
2. 在该子 GameObject 上添加 `FMODUnity.StudioEventEmitter` 组件，配置好脚步声事件引用；
3. 在该子 GameObject 上添加 `ProjectII.Character.FootstepAudio` 脚本；
4. 将同物体上的 `StudioEventEmitter` 拖入 `FootstepAudio` 的 `footstepEmitter` 字段；
5. 根据需要调整 `footstepDistanceInterval`（默认 0.75，即角色移动 0.75 个世界单位触发一次脚步声）。

注意：
- `Rigidbody2D` 和 `CharacterController` 不需要手动赋值，脚本会自动通过 `GetComponentInParent` 和 `GameSceneManager` 获取。
- 如果原先 `CharacterController` 上的 `footstepSFX_Emitter` 字段有序列化引用，该字段已被移除，旧引用会自动失效，无需手动处理。

## 2. 搭建 FMOD_FootStepPlayer_Parameter_Trigger

1. 在场景中创建一个空 GameObject，放置在需要切换地面材质的区域；
2. 添加一个 `BoxCollider2D`（或其他 2D 碰撞体），勾选 `Is Trigger`，调整大小覆盖目标区域；
3. 添加 `ProjectII.FX.FMOD_FootStepPlayer_Parameter_Trigger` 脚本；
4. 在 Inspector 中设置 `groundMaterial`（Wood = 0, Tile = 1，可扩展）；
5. 可选：调整 Gizmo 颜色以便在 Scene 视图中区分不同材质区域。

工作原理：
- 玩家进入触发器时，脚本通过 `CompareTag("Player")` 确认是玩家；
- 然后通过 `GameSceneManager.Instance.CurrentPlayerCharacter` 拿到玩家角色；
- 再通过 `GetComponentInChildren<FootstepAudio>()` 找到脚步声组件；
- 调用 `FootstepAudio.SetParameter("GroundMaterial", value)` 设置参数。

注意：
- 触发器区域应当互相覆盖或紧密相邻，确保玩家始终处于某个触发器内，不会出现"无材质"的空白区域。
- 玩家离开触发器时不需要重置参数，由下一个触发器覆盖设置即可。
- 触发器在 Scene 视图中始终绘制 Gizmo（无论是否选中），方便关卡设计。

## 3. 整体层级结构示例

```
Player (Tag: Player, Layer: Player)
├── CharacterController 脚本
├── Rigidbody2D
├── Collider2D
├── Hotbar 脚本
├── Backpack 脚本
├── FootstepAudio 子物体
│   ├── FootstepAudio 脚本
│   └── FMODUnity.StudioEventEmitter（脚步声事件）
└── ...
```

## 4. 调用关系总结

```
FMOD_FootStepPlayer_Parameter_Trigger
  │  OnTriggerEnter2D（玩家进入区域）
  │
  ├─→ GameSceneManager.Instance.CurrentPlayerCharacter
  │     └─→ GetComponentInChildren<FootstepAudio>()
  │           └─→ FootstepAudio.SetParameter("GroundMaterial", value)
  │                 └─→ StudioEventEmitter.SetParameter(...)
  │
FootstepAudio
  │  FixedUpdate（每物理帧）
  │
  ├─→ CharacterController.CurrentSpeedState（判断是否静止）
  ├─→ Rigidbody2D.velocity.magnitude（获取移动速度）
  └─→ StudioEventEmitter.Play()（达到距离阈值时播放）
```

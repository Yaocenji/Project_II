
# 脚本 ProjectII.Item.WorldItem
继承：ProjectII.SceneItems.InteractableBase

世界中的掉落物脚本。挂载在掉落物 Prefab 上，负责在场景中表现物品的掉落状态，并响应玩家的拾取交互。

WorldItem 与手持物（Item.Base）是**独立的两个 GameObject**：
- WorldItem 负责"物品在世界中"的状态：掉落外观、物理、可拾取范围；
- Item.Base 负责"物品在玩家手中"的状态：持握外观、使用逻辑。

## 成员 手持物 Prefab 引用 heldItemPrefab

GameObject 引用，指向对应的手持物 Prefab（挂有 Item.Base 派生类的预制体）。
可由关卡设计者在 Inspector 中直接指定，或由背包丢弃逻辑在实例化时赋值。

## 交互范围检测

通过 OnTriggerEnter2D / OnTriggerExit2D 检测玩家（Tag: "Player"）进出范围，
分别调用父类的 EnterInteractableRange / ExitInteractableRange。

## 方法 OnInteract（重写自 InteractableBase）

玩家按下交互键时调用。

1. 根据 heldItemPrefab 实例化对应的手持物 GameObject；
2. 通过 Player 标签找到玩家身上的 Backpack 组件；
3. 调用 Backpack.PutItemAuto 尝试放入背包；
4. 放入成功：销毁自身 GameObject；
5. 放入失败（背包已满等）：销毁刚实例化的手持物，保持自身不变。

## 设计说明

WorldItem 有 Collider（触发拾取范围）、掉落专用 Sprite、Rigidbody2D 等组件；
手持物（Item.Base）有持握动画、攻击逻辑等组件。
两者职责完全不同，强行复用同一 GameObject 会导致组件相互干扰。

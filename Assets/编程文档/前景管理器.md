
# 脚本 ProjectII.Render.ForegroundObject
# 脚本 ProjectII.Render.ForegroundManager
前景物品管理器

---

## 设计背景

游戏场景是 90° 纯俯视角，摄像机使用正交投影，没有真正的透视，所以理论上不论场景物体的高度是多少，都不会有近大远小的效果。

但是为了沉浸感和画面视觉效果，前景物体（离地面有一定高度的物体）具备以下三种效果，全部由物体的**虚拟高度**（`virtualHeight`，距地面的逻辑距离，非真实 z 轴）驱动：

1. **Scale 放大**：模拟物体距摄像机更近，因此看起来更大；
2. **高斯模糊**：模拟景深效果，高度越高模糊越强；
3. **XY 位置偏移**：渲染位置相对于逻辑位置向外发散偏移，方向为从摄像机位置指向物体的方向，模拟视差/透视感。

此外，`transform.z` 会被程序化地同步为 `virtualHeight`，使遮挡关系自动正确，无需手动设置渲染优先级。手动修改 `transform.z` 是不优雅的，但通过程序自动同步是可接受的。

---

## 系统架构

| 脚本 | 职责 |
|---|---|
| `ForegroundManager` | 场景单例，持有全局参数（摄像机、像素密度、效果幅度），每帧遍历所有注册的 ForegroundObject 并驱动效果更新 |
| `ForegroundObject` | 挂载在单个前景物体上，封装效果的具体应用逻辑，在 `OnEnable`/`OnDisable` 时向 Manager 注册/反注册 |

**注册流程**：
- `ForegroundObject.OnEnable` → 调用 `ForegroundManager.Instance.Register(this)`
- `ForegroundObject.OnDisable` → 调用 `ForegroundManager.Instance.Unregister(this)`
- `ForegroundManager.Start` → 调用 `FindObjectsByType` 主动扫描场景中已有的 ForegroundObject，解决启动顺序导致的未注册问题

## ForegroundManager Inspector 参数

| 参数 | 说明 |
|---|---|
| `mainCamera` | 场景主摄像机（若为空，Awake/Start 自动查找 `Camera.main`） |
| `pixelsPerUnit` | 素材的像素密度（ppu），用于计算全分辨率缩放倍数 |
| `maxHeight` | 虚拟高度归一化基准：高度达到此值时效果为最大值 |
| `maxScaleAddition` | 高度为 `maxHeight` 时的 scale 额外增量（如 0.5 表示最大放大到 1.5 倍） |
| `maxBlurRadius` | 高度为 `maxHeight` 时的模糊半径（全分辨率像素单位） |
| `maxPositionOffset` | 偏移缩放系数：高度为 `maxHeight` 时，距摄像机 1 个世界单位的物体偏移 `maxPositionOffset` 个世界单位 |

## 三种效果的计算细节

### 公共归一化参数 t

三种效果共享同一个线性归一化值：

```
t = Clamp01(height / maxHeight)
```

t 取值范围 [0, 1]，height = 0 时 t = 0（无效果），height = maxHeight 时 t = 1（最大效果）。

---

### 效果一：Scale 放大

```
scale = 1 + t × maxScaleAddition
```

应用方式：`transform.localScale = baseLocalScale × scale`

`baseLocalScale` 在 `OnEnable` 时记录，保留设计师在 Inspector 中设置的原始 scale，不被程序覆盖。

---

### 效果二：高斯模糊

#### 模糊半径计算

```
blurRadius（全分辨率像素）= t × maxBlurRadius
```

#### 全分辨率原则

模糊纹理的分辨率不基于精灵图的原始像素尺寸，而是基于其在屏幕上的**实际渲染像素尺寸**：

```
fullResScale = Screen.height / (camera.orthographicSize × 2 × pixelsPerUnit)
```

示例：摄像机 orthographicSize = 4，屏幕高度 1280，ppu = 32：
```
fullResScale = 1280 / (4 × 2 × 32) = 5
```
即 1 个像素风像素 = 5×5 真实像素。一张 16×16 的精灵图在全分辨率下为 80×80。

#### 模糊纹理生成流程

> 在 `ForegroundObject.GenerateBlurTexture` 中执行，含脏检查：blurRadius 变化 ≥ 0.5px 或 fullResScale 变化时才重新生成。

1. 从图集读取原始像素（`sprite.textureRect` 定位区域，需要纹理开启 **Read/Write Enabled**）
2. 双线性插值缩放到全分辨率（`srcW × fullResScale`, `srcH × fullResScale`）
3. 创建尺寸为 `(fullResW + 2r) × (fullResH + 2r)` 的缓冲区，四周填充透明（r = 模糊半径整数值）——模糊会向外扩散边界，需要预留空间
4. 可分离高斯卷积：先水平方向，再垂直方向（kernel 大小 `2r+1`，σ = r/3）
5. 生成 `Texture2D`（RGBA32，保留 alpha，Clamp 模式）
6. 计算新 pivot：`pivotX = (origPivot.x × fullResScale + r) / finalW`（加上 padding 偏移量）
7. 创建新 `Sprite`，PPU = `origPpu × fullResScale`（保持世界空间物理尺寸不变）
8. 赋给 `SpriteRenderer.sprite`

> 模糊半径 = 0 时直接恢复原始精灵，不生成纹理。`OnDisable` 时同样恢复原始精灵并销毁运行时纹理。

### 效果三：XY 位置偏移

#### 当前实现（距离线性模式）

```
direction = objBasePos - camWorldPos
offset = direction × t × maxPositionOffset
```

**含义**：偏移方向从摄像机指向物体（向外发散），偏移幅度同时正比于高度（通过 t）和物体到摄像机的距离（通过未归一化的 direction）。距摄像机越远的物体偏移越大，与真实视差行为一致。

`maxPositionOffset` 的物理含义：高度为 `maxHeight` 时，距摄像机 **1 个世界单位**的物体偏移 **`maxPositionOffset` 个世界单位**。

#### 与透视正确公式的关系

物理上精确的视差公式为：

```
offset = (objBasePos - camWorldPos) × height / (H - height)
≈ (objBasePos - camWorldPos) × (height / H)    // 当 height << H 时线性近似
```

当前实现即为此线性近似，令 `k = maxPositionOffset / maxHeight`，则：

```
offset = direction × (height / maxHeight) × maxPositionOffset
       = direction × height × k
```

与近似公式完全等价（等效摄像机高度 H = 1/k），无需额外参数。

#### 增量更新与 BaseWorldPosition

由于偏移每帧都在修改 `transform.position`，直接读 `transform.position` 会包含上帧的偏移量，导致方向向量计算出现误差漂移。

解决方案：
- `ForegroundObject` 内部缓存上一帧的 `lastPositionOffset`
- 每帧通过**增量方式**应用偏移：`pos += newOffset - lastOffset`
- 对外暴露 `BaseWorldPosition = (Vector2)transform.position - lastPositionOffset`，供 Manager 计算方向时使用

---

## z 轴同步

```
transform.position.z = virtualHeight
```

每帧在 `UpdateScaleAndOffset` 中执行。Unity 2D 的 SpriteRenderer 在 z 值不同时自动处理遮挡排序，高度越高（z 越大）的物体会被渲染在更前面——这与前景物体"离摄像机更近"的语义一致。

---

## 使用要求

- 精灵纹理必须在 Unity 纹理导入设置中开启 **Read/Write Enabled**，否则 CPU 无法读取像素，模糊纹理将无法生成（脚本会打印 Warning）。
- `ForegroundManager` 必须作为场景中的 GameObject 存在（`[DefaultExecutionOrder(-98)]`，在大多数脚本之前执行）。
- 若场景中不存在 `ForegroundManager`，`ForegroundObject` 的效果不会生效（注册失败时不报错，物体保持原始状态）。

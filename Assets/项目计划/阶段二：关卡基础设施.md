
# 阶段二：关卡基础设施与演出工具

> 预计工期：1~2 周
> 前置依赖：阶段一（可玩性闭环）完成
> 核心目标：搭建关卡设计所需的工具链，让"搭关卡"变成"拼积木"

---

## 为什么需要这个阶段？

阶段一完成后，你有了一个能打能死的战斗沙盒。
但要把它变成一个**有节奏、有叙事、有演出**的关卡，还需要以下工具：

- 怎么在玩家走到某个位置时触发事件？→ **关卡事件系统**
- 怎么让 NPC 说话、推进剧情？→ **对话系统**
- 怎么从一个场景切换到另一个？→ **场景切换**
- 怎么让 BGM 随关卡进程变化？→ **BGM 管理**
- 怎么做开场镜头、Boss 登场等演出？→ **镜头演出工具**

这些是关卡设计师（也就是你自己）的"画笔"，没有它们，关卡搭建会非常痛苦。

## 任务清单

### 1. 关卡事件 / 脚本系统 🟡 P1

#### 1.1 TriggerZone 触发区域

**目标**：一个通用的触发区域组件，玩家进入/停留/离开时触发事件。

**设计要点**：
- 创建 `ProjectII.Level.TriggerZone`（MonoBehaviour）
- 成员：
  - `triggerOnce`：是否只触发一次（bool，默认 true）
  - `triggerTag`：触发对象的 Tag（string，默认 "Player"）
  - `onEnter`：UnityEvent，进入时触发
  - `onStay`：UnityEvent，停留时触发（可选，慎用）
  - `onExit`：UnityEvent，离开时触发
- 使用 `OnTriggerEnter2D` / `OnTriggerExit2D` 检测
- 触发一次后如果 `triggerOnce = true`，禁用自身
- 在 Scene 视图中绘制 Gizmo（参考已有的 FMOD_FootStepPlayer_Parameter_Trigger）

**放置位置**：`Assets/Level/TriggerZone.cs`
**命名空间**：`ProjectII.Level`

#### 1.2 LevelEvent 关卡事件

**目标**：一组可复用的关卡事件脚本，挂载在 TriggerZone 上或由其他方式调用。

**常用事件类型**（每个一个小脚本）：

| 事件 | 功能 | 说明 |
|------|------|------|
| `SpawnEnemies` | 在指定位置生成敌人 | 引用敌人 Prefab + 生成点 Transform 列表 |
| `LockDoor` / `UnlockDoor` | 锁门 / 开门 | 引用 Door1 组件，控制其可交互状态 |
| `PlayDialogue` | 播放对话 | 引用对话数据，启动对话系统 |
| `PlayBGM` | 切换 BGM | 引用 FMOD 事件 |
| `CameraMoveTo` | 镜头移动到指定位置 | 引用目标 Transform，触发镜头演出 |
| `SetCheckpoint` | 设置重生点 | 记录当前位置为检查点 |
| `LoadScene` | 加载场景 | 指定目标场景名 |
| `WaitAndDo` | 延迟执行 | 等待 N 秒后触发另一个 UnityEvent |
| `DestroyTarget` | 销毁指定对象 | 引用目标 GameObject |

**设计原则**：
- 每个事件脚本尽量小而专一（单一职责）
- 通过 UnityEvent 串联，在 Inspector 中可视化配置
- 关卡设计者不需要写代码，只需要在 Inspector 中拖拽配置

**放置位置**：`Assets/Level/Events/` 目录下
**命名空间**：`ProjectII.Level`

#### 1.3 一次性 Gameplay 脚本

**目标**：为特定关卡编写的一次性脚本，处理该关卡独有的逻辑。

**设计原则**：
- 命名格式：`Level{场景名}_{功能描述}.cs`，例如 `LevelScene0_OpeningCutscene.cs`
- 可以直接引用场景中的具体对象
- 不需要通用性，只服务于当前关卡
- 通过 TriggerZone 或 Start() 触发

**放置位置**：`Assets/Scenes/{场景名}/场景代码/` 目录下（已有此结构）

### 2. 对话系统 🟡 P1

#### 2.1 对话数据格式

**目标**：定义对话数据的存储格式。

**推荐方案**：ScriptableObject

```
DialogueData (ScriptableObject)
├── dialogueID: string
├── lines: DialogueLine[]
│   ├── speaker: string（说话者名称）
│   ├── portrait: Sprite（头像，可选）
│   ├── text: string（对话文本）
│   ├── autoAdvance: bool（是否自动推进）
│   └── autoAdvanceDelay: float（自动推进延迟）
└── onComplete: UnityEvent（对话结束后触发）
```

**放置位置**：`Assets/Dialogue/DialogueData.cs`
**命名空间**：`ProjectII.Dialogue`

#### 2.2 对话管理器

**目标**：控制对话的播放流程。

**设计要点**：
- 创建 `ProjectII.Dialogue.DialogueManager`（单例）
- 方法：
  - `StartDialogue(DialogueData data)`：开始播放对话
  - `AdvanceLine()`：推进到下一行
  - `SkipLine()`：跳过当前行的打字机效果
  - `EndDialogue()`：结束对话
- 对话播放时：
  - 禁用玩家输入（通过 InputManager 禁用 ActionMap）
  - 显示对话 UI
  - 打字机效果逐字显示文本
  - 玩家按交互键推进 / 跳过
- 对话结束时：
  - 恢复玩家输入
  - 隐藏对话 UI
  - 触发 `onComplete` 事件

**放置位置**：`Assets/Dialogue/DialogueManager.cs`

#### 2.3 对话 UI

**目标**：对话框的 UI 表现。

**内容**：
- 对话框背景（底部或中下方）
- 说话者名称
- 对话文本（打字机效果）
- 头像（可选）
- 推进提示（闪烁的箭头或文字）

**第一版用默认 UI 即可，后期再美化。**

### 3. 场景切换 / 加载 🟡 P1

#### 3.1 SceneLoader 场景加载器

**目标**：统一管理场景的异步加载和过渡效果。

**设计要点**：
- 创建 `ProjectII.Manager.SceneLoader`（单例，DontDestroyOnLoad）
- 方法：
  - `LoadScene(string sceneName)`：异步加载场景
  - `LoadScene(string sceneName, string spawnPointID)`：加载场景并指定出生点
- 过渡效果：
  - 淡入黑屏 → 异步加载 → 淡出黑屏
  - 使用 Canvas + CanvasGroup 的 alpha 动画
- 加载完成后：
  - 查找场景中的 SpawnPoint，将玩家移动到指定位置
  - 初始化场景管理器

**放置位置**：`Assets/Manager/SceneLoader.cs`
**命名空间**：`ProjectII.Manager`

#### 3.2 SceneTransitionTrigger 场景切换触发器

**目标**：玩家走到特定区域时触发场景切换。

- 继承或配合 TriggerZone 使用
- 指定目标场景名和出生点 ID
- 可以设置切换条件（如：需要持有某物品）

#### 3.3 SpawnPoint 出生点

**目标**：标记场景中的出生位置。

- 简单组件，只有一个 `spawnPointID`（string）
- 在 Scene 视图中绘制 Gizmo 标记

### 4. BGM / 环境音管理 🟠 P2

#### 4.1 MusicManager 音乐管理器

**目标**：管理 BGM 的播放、切换、淡入淡出。

**设计要点**：
- 创建 `ProjectII.Manager.MusicManager`（单例，DontDestroyOnLoad）
- 使用 FMOD 的 StudioEventEmitter 或直接操作 EventInstance
- 方法：
  - `PlayBGM(string eventPath)`：播放 BGM（带淡入）
  - `StopBGM(float fadeOutTime)`：停止 BGM（带淡出）
  - `SetParameter(string name, float value)`：设置 BGM 参数（用于动态音乐）
- 支持 BGM 层叠（FMOD 的参数驱动，如战斗强度参数）

**放置位置**：`Assets/Manager/MusicManager.cs`
**命名空间**：`ProjectII.Manager`

#### 4.2 环境音区域

**目标**：玩家进入不同区域时切换环境音。

- 复用 TriggerZone + FMOD Snapshot 机制（已有 FMOD_Snapshot_Trigger）
- 可能需要扩展支持多层环境音叠加

**备注**：FMOD 项目中已有丰富的 Ambiance 素材（洞穴/蝉鸣/雨声/风声/城市等），可以直接使用。

### 5. 镜头演出工具 🟠 P2

#### 5.1 CameraCutscene 镜头过场

**目标**：在特定时刻控制镜头移动到指定位置，用于演出。

**设计要点**：
- 利用 Cinemachine 的 Virtual Camera 优先级切换
- 创建 `ProjectII.FX.CameraCutscene`（MonoBehaviour）
- 方法：
  - `PlayCutscene()`：开始过场
    1. 禁用玩家输入
    2. 激活过场用的 Virtual Camera（高优先级）
    3. 镜头平滑移动到目标位置
    4. 停留指定时间
    5. 镜头返回玩家
    6. 恢复玩家输入
- 支持多段路径（Cinemachine Path 或 DOTween）
- 通过 UnityEvent 在过场的关键帧触发事件（如：镜头到达 Boss 位置时播放 Boss 吼叫音效）

**放置位置**：`Assets/FX/CameraCutscene.cs`
**命名空间**：`ProjectII.FX`

## 验收标准

完成阶段二后，应该能在 Inspector 中**纯拖拽配置**实现以下关卡流程：

```
玩家出生在 SpawnPoint
  → 走到 TriggerZone A → 触发对话（NPC 说话）
  → 走到 TriggerZone B → 锁门 + 刷出敌人
  → 消灭所有敌人 → 开门
  → 走到 TriggerZone C → 镜头演出（展示 Boss 区域）
  → 走到 TriggerZone D → BGM 切换为战斗音乐
  → 走到场景出口 → 加载下一个场景
```

```
Day 1-2:  TriggerZone + 基础 LevelEvent 脚本

```

## 注意事项

1. **UnityEvent 是你的好朋友**：关卡事件系统的核心思路是"触发器 + UnityEvent"，让关卡设计在 Inspector 中完成，而不是写代码。
2. **对话系统不要过度设计**：第一版不需要分支对话、选项、条件判断。线性对话就够了。后期如果需要复杂对话，再考虑 Yarn Spinner 等方案。
3. **场景切换要考虑 DontDestroyOnLoad**：SceneLoader 和 MusicManager 需要跨场景存活。注意与现有的 GameSceneManager（场景级单例）区分。
4. **镜头演出可以用 Cinemachine Timeline**：如果你熟悉 Unity Timeline，可以直接用 Timeline + Cinemachine Track 做镜头演出，比自己写代码更直观。

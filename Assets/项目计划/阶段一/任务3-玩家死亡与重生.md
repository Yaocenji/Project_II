# 任务 3：玩家死亡与重生

> 优先级：🔴 P0
> 预计工期：0.5~1 天
> 前置依赖：任务 1（Damageable 组件）
> 命名空间：`ProjectII.Combat`（死亡处理）、`ProjectII.FX`（视觉效果）
> 文件位置：`Assets/Combat/`、`Assets/FX/`

## 一、概述

本任务的目标是处理玩家 HP 归零后的完整流程：从受击反馈到死亡演出再到场景重载。虽然第一版的核心逻辑很简单（HP=0 → 重载场景），但良好的死亡反馈是游戏手感的重要组成部分。

## 二、玩家 Damageable 配置

### 2.1 挂载位置

在玩家角色 Prefab 的根 GameObject 上挂载 `Damageable` 组件。

```
Player GameObject (Layer: Player)
├── CharacterController
├── Damageable              ← 新增
│   ├── maxHP: 100
│   └── invincibleDuration: 0.5  // 玩家无敌帧比敌人长，给喘息空间
├── PlayerDeathHandler      ← 新增
├── DamageFlash             ← 新增（任务 1 的受击闪白）
├── DamageScreenEffect      ← 新增（受击屏幕效果）
└── ... 其他现有组件
```

### 2.2 建议数值

| 参数 | 值 | 设计意图 |
|------|-----|----------|
| maxHP | 100 | 整数好算，方便调试 |
| invincibleDuration | 0.5 秒 | 比敌人的 0.1 秒长，防止玩家被围殴秒杀 |

## 三、PlayerDeathHandler 组件

### 3.1 组件职责

专门处理玩家死亡逻辑的组件，订阅 `Damageable` 的事件。

### 3.2 成员变量

```
[Header("死亡设置")]
[SerializeField] deathDelay      : float   // 死亡后到重载场景的延迟（秒），默认 1.5
[SerializeField] slowMotionScale : float   // 死亡慢动作时间缩放，默认 0.3

// 组件引用
damageable       : Damageable              // 缓存引用
characterController : CharacterController  // 缓存引用（用于禁用输入）
```

### 3.3 死亡处理流程

```
OnDeath():  // 由 Damageable.OnDeath 事件触发
│
├─ 1. 禁用玩家输入
│     characterController.enabled = false
│     // 或者禁用 InputManager
│
├─ 2. 停止玩家移动
│     rb.velocity = Vector2.zero
│
├─ 3. 播放死亡效果（详见第四节）
│     StartCoroutine(DeathSequence())
│
└─ DeathSequence():
      ├─ 触发慢动作（可选）
      │   Time.timeScale = slowMotionScale
      │
      ├─ 触发屏幕效果（黑屏/红屏）
      │
      ├─ yield return WaitForSecondsRealtime(deathDelay)
      │   // 注意：用 Realtime，因为 timeScale 被改了
      │
      ├─ 恢复时间缩放
      │   Time.timeScale = 1f
      │
      └─ 重载场景
          SceneManager.LoadScene(SceneManager.GetActiveScene().name)
```

## 四、可做的死亡/受击效果及技术实现路径

以下列出从简到繁的各种效果，标注了优先级和实现难度。**第一版只需实现 P0 的效果**，其余作为后续迭代的参考。

### 4.1 受击屏幕闪红 🔴 P0

**效果**：玩家被击中时，屏幕边缘闪红色，提示受到伤害。

**技术方案 A：UI Image 叠加层（推荐第一版）**

最简单的方案，零额外依赖。

```
实现步骤：
1. 在 Canvas 上创建一个全屏 Image（命名为 DamageOverlay）
2. Image 使用一张从边缘到中心渐变透明的红色纹理
   （或者直接用纯红色 Image，中心挖空）
3. 默认 alpha = 0（不可见）
4. 创建 DamageScreenEffect 组件，订阅 Damageable.OnDamaged
5. 受击时：alpha 从 0.4 → 0 做 Lerp（持续 0.3 秒）
```

```
DamageScreenEffect 组件：
├── [SerializeField] damageOverlayImage : Image
├── [SerializeField] flashAlpha : float = 0.4    // 闪红最大透明度
├── [SerializeField] flashDuration : float = 0.3 // 闪红持续时间
│
└── OnDamaged(damage, source):
    StopAllCoroutines()  // 防止多次受击叠加
    StartCoroutine(FlashCoroutine())
    
    FlashCoroutine():
        设置 Image.color.a = flashAlpha
        float t = 0
        while (t < flashDuration):
            t += Time.deltaTime
            Image.color.a = Lerp(flashAlpha, 0, t / flashDuration)
        Image.color.a = 0
```

**技术方案 B：Post Processing Volume（后期可选）**

使用 URP 的 Volume + Color Adjustments 或 Vignette 效果。

```
实现思路：
1. 创建一个 Volume（Global），添加 Vignette 效果
2. 默认 Vignette intensity = 0
3. 受击时：
   - Vignette intensity 从 0.6 → 0 做 Lerp
   - Vignette color 设为红色
4. 需要通过代码控制 Volume 参数：
   volume.profile.TryGet<Vignette>(out var vignette)
   vignette.intensity.Override(value)
```

优点：效果更自然，与场景光照融合
缺点：需要 URP Volume 配置，稍复杂

### 4.2 受击镜头抖动 🔴 P0

**效果**：玩家被击中时镜头抖动，增强打击感。

**技术方案：复用现有 CinemachineShake**

项目已有完整的 `CinemachineShake` 组件（支持随机震动 + 定向后坐力），直接复用。

```
实现步骤：
1. 在 PlayerDeathHandler 或 DamageScreenEffect 中订阅 OnDamaged
2. 受击时调用：
   var shake = FindObjectOfType<CinemachineShake>();
   // 参数：随机震动强度, 后坐力方向（从伤害源指向玩家）, 后坐力力度
   Vector3 recoilDir = (transform.position - source.transform.position).normalized;
   shake.Shake(0.2f, recoilDir, 0.15f);
```

**CinemachineShake.Shake() 接口回顾：**
```csharp
public void Shake(float shakeForce, Vector3 recoilDir, float recoilForce)
```
- `shakeForce`：随机震动强度（Perlin Noise 驱动）
- `recoilDir`：定向后坐力方向
- `recoilForce`：后坐力推力

**建议参数：**
- 普通受击：`Shake(0.2f, recoilDir, 0.15f)`
- 重击/死亡：`Shake(0.5f, recoilDir, 0.3f)`

### 4.3 死亡慢动作 🟡 P1

**效果**：玩家死亡瞬间时间变慢，营造戏剧感。

**技术方案：Time.timeScale**

```
实现步骤：
1. 死亡时：Time.timeScale = 0.3（30% 速度）
2. 持续 0.5~1 秒（用 WaitForSecondsRealtime）
3. 恢复：Time.timeScale = 1.0

注意事项：
- 使用 WaitForSecondsRealtime 而非 WaitForSeconds（后者受 timeScale 影响）
- Time.fixedDeltaTime 也需要同步调整：
  Time.fixedDeltaTime = 0.02f * Time.timeScale
- 恢复时记得还原 fixedDeltaTime
```

**风险**：timeScale 会影响所有系统（物理、动画、协程），需要确保不会产生副作用。第一版可以不做。

### 4.4 死亡黑屏过渡 🟡 P1

**效果**：死亡后屏幕渐变为黑色，然后重载场景。

**技术方案：UI Image 渐变**

```
实现步骤：
1. 在 Canvas 上创建一个全屏黑色 Image（命名为 FadeOverlay）
2. 默认 alpha = 0
3. 死亡时：alpha 从 0 → 1 做 Lerp（持续 1 秒）
4. 完全黑屏后重载场景

FadeToBlack():
    float duration = 1.0
    float t = 0
    while (t < duration):
        t += Time.unscaledDeltaTime  // 不受 timeScale 影响
        fadeImage.color = new Color(0, 0, 0, t / duration)
    
    // 完全黑屏后重载
    SceneManager.LoadScene(...)
```

## 六、第一版最小实现方案

只做 P0 的内容，预计 0.5~1 天完成：

```
玩家被击中
├─ Damageable.TakeDamage() 扣血
├─ OnDamaged 事件触发：
│   ├─ DamageFlash：Sprite 闪白
│   ├─ DamageScreenEffect：屏幕边缘闪红
│   └─ CinemachineShake：镜头抖动
│
├─ if HP > 0：继续战斗
│
└─ if HP <= 0：
    └─ OnDeath 事件触发：
        └─ PlayerDeathHandler：
            ├─ 禁用玩家输入
            ├─ 停止移动
            ├─ 等待 1.5 秒
            └─ SceneManager.LoadScene(当前场景)
```

## 九、注意事项

1. **WaitForSecondsRealtime**：如果做了慢动作，所有等待都要用 Realtime 版本
2. **Time.timeScale 恢复**：场景重载前必须恢复 `Time.timeScale = 1f`，否则新场景也会是慢动作
3. **输入禁用方式**：建议禁用 `CharacterController` 组件而非 InputManager，因为 InputManager 是全局单例
4. **Canvas 层级**：DamageOverlay 和 FadeOverlay 应该在 UI 最顶层（Sort Order 最高），确保覆盖所有 UI
5. **不要做检查点/存档**：第一版直接重载整个场景，检查点系统是后期功能

# 任务 1：生命值 / 伤害系统

> 优先级：🔴 P0
> 预计工期：1.5~2 天
> 前置依赖：无
> 命名空间：`ProjectII.Combat`
> 文件位置：`Assets/Combat/`

---

## 一、概述

本任务的目标是建立一套通用的生命值/伤害系统，使得任何可被伤害的对象（玩家、敌人、可破坏物）都能接收伤害、触发反馈、并在 HP 归零时死亡。同时将现有的子弹命中逻辑接入伤害系统，打通"射击→造成伤害"这条关键链路。

## 二、Damageable 组件

### 2.1 组件定位

`Damageable` 是一个**独立的 MonoBehaviour 组件**，采用组合模式（而非继承），任何需要被伤害的 GameObject 只需挂载此组件即可获得生命值管理能力。

### 2.2 成员变量

```
[Header("生命值")]
[SerializeField] maxHP          : int       // 最大生命值，Inspector 可调
                 currentHP      : int       // 当前生命值（运行时）

[Header("无敌帧")]
[SerializeField] invincibleDuration : float // 受击后无敌持续时间（秒），默认 0.2
                 isInvincible       : bool  // 当前是否处于无敌状态（运行时）
```

### 2.3 公有属性（只读，供 HUD 等外部读取）

```
CurrentHP   : int   { get; }    // 当前 HP
MaxHP       : int   { get; }    // 最大 HP
IsAlive     : bool  { get; }    // currentHP > 0
IsInvincible: bool  { get; }    // 当前是否无敌
HPRatio     : float { get; }    // currentHP / maxHP，用于血条 UI 填充
```

### 2.4 事件系统

采用 **C# event + UnityEvent 双轨制**：
- C# event：给代码逻辑订阅（性能好，编译期检查）
- UnityEvent：给 Inspector 拖拽用（策划友好，可视化配置反馈）

```
// C# 事件
event System.Action<int, GameObject>  OnDamaged_CSharp    // (damage, source)
event System.Action                   OnDeath_CSharp
event System.Action<int, int>         OnHPChanged_CSharp  // (currentHP, maxHP)

// UnityEvent（Inspector 可拖拽）
UnityEvent<int, GameObject>  OnDamaged_Unity     // (damage, source)
UnityEvent                   OnDeath_Unity
UnityEvent<int, int>         OnHPChanged_Unity   // (currentHP, maxHP)
```

### 2.5 公有方法

#### `TakeDamage(int damage, GameObject source)`

受到伤害的核心方法。

**执行流程：**
```
TakeDamage(damage, source)
│
├─ if (isInvincible || !IsAlive) → return，忽略伤害
│
├─ currentHP = Mathf.Max(0, currentHP - damage)
│
├─ 触发 OnDamaged 事件（C# + Unity 双轨）
├─ 触发 OnHPChanged 事件（C# + Unity 双轨）
│
├─ if (currentHP <= 0)
│   └─ 触发 OnDeath 事件（C# + Unity 双轨）
│
└─ if (invincibleDuration > 0)
    └─ 启动无敌帧协程 InvincibleCoroutine()
```

#### `Heal(int amount)`

恢复生命值。

**执行流程：**
```
Heal(amount)
│
├─ if (!IsAlive) → return，死亡后不能回血
│
├─ currentHP = Mathf.Min(maxHP, currentHP + amount)
│
└─ 触发 OnHPChanged 事件
```

#### `ResetHP()`

重置生命值到最大值（用于重生/场景重载）。

### 2.6 无敌帧协程

```
InvincibleCoroutine():
    isInvincible = true
    yield return WaitForSeconds(invincibleDuration)
    isInvincible = false
```

注意：如果在无敌帧期间再次受击，不会重置无敌帧计时器（因为 TakeDamage 入口就被 isInvincible 拦截了）。

### 2.7 生命周期

```
Awake():
    currentHP = maxHP

OnDestroy():
    清空所有事件订阅（防止内存泄漏）
```

## 三、子弹命中接入伤害

### 3.1 当前现状

`BulletRevolver.Hit()` 目前只有 Debug.Log：

```csharp
protected override void Hit(GameObject hitTarget, Vector2 hitPoint, Vector2 hitNormal)
{
    hitSomething = true;
    this.hitPoint = hitPoint;
    this.hitNormal = hitNormal;
    Debug.Log("左轮击中, hitTarget: " + hitTarget.name + ...);
}
```

### 3.2 改造方案

#### 方案 A：在 BulletBase 中统一处理（推荐）

在 `BulletBase` 中新增 `damage` 字段，并在 `PerformRaycast()` 和 `OnTriggerEnter2D()` 的 Hit 调用后，自动尝试对目标造成伤害。

**新增字段：**
```
[Header("Damage")]
[SerializeField] protected int damage = 1;  // 子弹伤害值
```

**修改 PerformRaycast()：**
在调用 `Hit()` 之后，追加伤害逻辑：
```
if (hit.collider != null)
{
    Hit(hit.collider.gameObject, hit.point, hit.normal);
    // 新增：尝试对目标造成伤害
    ApplyDamage(hit.collider.gameObject);
}
```

**新增 ApplyDamage() 方法：**
```
protected virtual void ApplyDamage(GameObject target)
{
    Damageable damageable = target.GetComponent<Damageable>();
    if (damageable != null)
    {
        damageable.TakeDamage(damage, Shooter);
    }
}
```

**同样修改 OnTriggerEnter2D()：**
在调用 `Hit()` 之后追加 `ApplyDamage(other.gameObject)`。

#### 方案 B：在子类中各自处理

在 `BulletRevolver.Hit()` 中直接调用 `TakeDamage`。灵活但容易遗漏。

**推荐方案 A**，因为伤害是所有子弹的共性行为。

### 3.3 伤害值来源

第一版：子弹自身携带 `damage` 字段，在 Inspector 中配置。
后期可扩展：从武器数据 ScriptableObject 中读取，支持伤害加成/暴击等。

### 3.4 Revolver 具体改动

`Revolver` 的子弹池创建时，`BulletRevolver` 的 `damage` 字段由 Prefab 上的 Inspector 值决定。
第一版左轮伤害建议值：**每发 25 点**（配合敌人 100 HP，4 枪击杀，6 发弹匣有容错）。

## 四、受击反馈系统

受击反馈是**游戏手感的核心**。所有反馈都通过订阅 `Damageable.OnDamaged` 事件来触发，与伤害逻辑完全解耦。

### 4.1 受击闪白（Sprite Flash）

**目标**：被击中瞬间，Sprite 整体闪白 0.1 秒。

**技术方案：MaterialPropertyBlock**

不替换材质，而是通过 `MaterialPropertyBlock` 修改 shader 参数，避免打断 SRP Batcher。

**实现思路：**
```
创建 DamageFlash 组件（挂在有 SpriteRenderer 的对象上）
│
├─ Awake(): 获取 SpriteRenderer 引用，获取同对象的 Damageable 引用
├─ OnEnable(): 订阅 Damageable.OnDamaged_CSharp
├─ OnDisable(): 取消订阅
│
└─ OnDamaged():
    启动协程 FlashCoroutine():
        设置 MaterialPropertyBlock 的 _FlashAmount = 1（全白）
        Lerp 回 0（0.1 秒内）
        恢复原始状态
```

**Shader 侧**：
- 如果使用 URP 2D Lit Sprite Shader，可以通过 `_Color` 属性临时设为白色
- 更精细的方案：自定义 shader 添加 `_FlashAmount` 参数，在片元着色器中 lerp(原色, 白色, _FlashAmount)

**简化方案（第一版推荐）**：
直接修改 `SpriteRenderer.color` 为白色，然后 Lerp 回原色。虽然不如 shader 方案精确，但零额外开发成本。

```
FlashCoroutine():
    originalColor = spriteRenderer.color
    spriteRenderer.color = Color.white
    float t = 0
    while (t < flashDuration):
        t += Time.deltaTime
        spriteRenderer.color = Color.Lerp(Color.white, originalColor, t / flashDuration)
    spriteRenderer.color = originalColor
```

### 4.2 受击音效

**目标**：被击中时播放受击音效。

**技术方案：**
- 创建 `DamageAudio` 组件
- 持有 `FMODUnity.EventReference` 引用（受击音效事件）
- 订阅 `Damageable.OnDamaged_CSharp`
- 受击时调用 `FMODUnity.RuntimeManager.PlayOneShot(hitSFX, transform.position)`

**音效设计建议：**
- 玩家受击：沉闷的肉体撞击声 + 轻微的喘息声
- 敌人受击：清脆的打击声（给玩家正反馈）
- 可以在 FMOD 中用 Random Container 做多变体，避免重复感

### 4.3 击退效果（可选，建议预留接口）

**目标**：被击中时向远离伤害源的方向推一下。

**技术方案：**
```
创建 DamageKnockback 组件
│
├─ [SerializeField] knockbackForce : float  // 击退力度
│
└─ OnDamaged(damage, source):
    Vector2 direction = (transform.position - source.transform.position).normalized
    rb.AddForce(direction * knockbackForce, ForceMode2D.Impulse)
```

**第一版建议**：先不实现，但在 `Damageable.OnDamaged` 事件中已经传递了 `source` 参数，后期可以直接接入。

### 4.4 受击粒子特效（可选）

**目标**：被击中位置产生粒子效果（血花/火花）。

**技术方案：**
- 在 `BulletRevolver.Hit()` 中已有 `hitPoint` 和 `hitNormal`
- 可以在 Hit 时通过 `FXManager` 在 hitPoint 位置生成粒子
- 第一版可以用简单的 ParticleSystem Prefab

## 七、注意事项

1. **Damageable 不关心"谁"在用它**：玩家、敌人、可破坏物都用同一个组件，通过事件驱动不同的反馈
2. **事件解耦**：伤害逻辑（扣血）和反馈逻辑（闪白/音效/击退）完全分离，通过事件连接
3. **Layer 机制已完备**：`BulletBase` 已有完整的 Layer 自动设置逻辑（Player→FriendlyFire，Enemy→EnemyFire），不需要修改
4. **对象池兼容**：子弹从池中取出时已经调用了 `Init()`，damage 字段在 Prefab 上配置，池化不影响
5. **命名空间**：所有新文件使用 `ProjectII.Combat`，与项目规范一致

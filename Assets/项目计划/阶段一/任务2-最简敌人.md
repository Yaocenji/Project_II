> 优先级：🔴 P0

> 命名空间：`ProjectII.Enemy`

---

## 一、概述

本任务的目标是创建一个最简单的敌人系统：能被打死、能打玩家。不追求复杂的 AI 行为，只需要形成"敌人存在→玩家射击→敌人死亡"和"敌人发现玩家→追击→攻击玩家"的双向战斗循环。

**核心原则：先能跑能打，复杂 AI 留到后面。**

## 二、敌人基类 EnemyBase

### 2.1 组件架构

敌人 GameObject 的组件组成：

```
Enemy GameObject (Layer: Enemy)
├── SpriteRenderer          // 临时 Sprite（方块/圆形）
├── Rigidbody2D             // Dynamic, gravityScale=0
├── CircleCollider2D        // 碰撞体
├── Damageable              // 生命值组件（任务 1）
└── EnemyBase（或子类）      // AI 逻辑
```

### 2.2 状态机设计

采用**枚举驱动的简单状态机**，不使用行为树或复杂状态机框架。

#### 状态枚举

```
enum EnemyState
{
    Idle,       // 待机：站桩不动
    Chase,      // 追击：向玩家移动
    Attack,     // 攻击：对玩家造成伤害
    Dead        // 死亡：播放效果后销毁
}
```

#### 状态转换图

```
                    发现玩家（进入检测范围）
        ┌─────────────────────────────────────┐
        │                                     ▼
     [Idle] ◄──────────────────────────── [Chase]
        ▲        丢失玩家（超出检测范围）      │
        │                                     │ 进入攻击范围
        │                                     ▼
        │                                 [Attack]
        │                                     │
        │         目标离开攻击范围             │
        │    ┌────────────────────────────────┘
        │    ▼
        │  [Chase] ◄─── 继续追击
        │
        │              HP ≤ 0（任何状态均可触发）
        └──────────────────┐
                           ▼
                        [Dead]
```

### 2.3 成员变量

```
[Header("检测设置")]
[SerializeField] detectionRadius  : float   // 发现玩家的检测半径，默认 8
[SerializeField] loseTargetRadius : float   // 丢失目标的半径（应 > detectionRadius），默认 12
[SerializeField] detectionLayer   : LayerMask // 检测层（Player 层）

[Header("移动设置")]
[SerializeField] moveSpeed        : float   // 移动速度，默认 3
[SerializeField] rotationSpeed    : float   // 转向速度，默认 360（度/秒）

[Header("攻击设置")]
[SerializeField] attackRange      : float   // 攻击范围，默认 1.5
[SerializeField] attackDamage     : int     // 攻击伤害，默认 20
[SerializeField] attackInterval   : float   // 攻击间隔（秒），默认 1.0
[SerializeField] attackWindup     : float   // 攻击前摇时间（秒），默认 0.3

// 运行时状态
currentState     : EnemyState               // 当前状态
targetPlayer     : Transform                // 追击目标
lastAttackTime   : float                    // 上次攻击时间
damageable       : Damageable               // 缓存的 Damageable 引用
rb               : Rigidbody2D             // 缓存的刚体引用
spriteRenderer   : SpriteRenderer          // 缓存的渲染器引用
```

### 2.4 生命周期

```
Awake():
    缓存组件引用（Damageable, Rigidbody2D, SpriteRenderer）
    订阅 Damageable.OnDeath_CSharp → OnDeath()
    设置初始状态为 Idle

Start():
    从 GameSceneManager 获取玩家引用（用于追踪目标）

Update():
    if (currentState == Dead) return
    
    根据 currentState 执行对应的状态逻辑：
        Idle   → UpdateIdle()
        Chase  → UpdateChase()
        Attack → UpdateAttack()

FixedUpdate():
    if (currentState == Chase)
        执行物理移动

OnDestroy():
    取消订阅 Damageable 事件
```

### 2.5 各状态详细逻辑

#### Idle 状态

```
UpdateIdle():
    // 检测玩家是否进入检测范围
    Collider2D hit = Physics2D.OverlapCircle(position, detectionRadius, detectionLayer)
    
    if (hit != null):
        targetPlayer = hit.transform
        切换到 Chase 状态
```

**行为**：站桩不动，每帧做一次 OverlapCircle 检测。

**优化建议**：可以降低检测频率（每 0.2 秒一次），但第一版每帧检测即可。

#### Chase 状态

```
UpdateChase():
    if (targetPlayer == null):
        切换到 Idle
        return
    
    float distance = Vector2.Distance(position, targetPlayer.position)
    
    // 检查是否丢失目标
    if (distance > loseTargetRadius):
        targetPlayer = null
        切换到 Idle
        return
    
    // 检查是否进入攻击范围
    if (distance <= attackRange):
        切换到 Attack
        return
    
    // 朝向玩家
    Vector2 direction = (targetPlayer.position - position).normalized
    RotateTowards(direction)

FixedUpdate_Chase():
    // 直线移动向玩家（MoveTowards）
    Vector2 direction = (targetPlayer.position - position).normalized
    rb.velocity = direction * moveSpeed
```

**行为**：直线追击玩家，不做寻路。如果玩家跑出 loseTargetRadius 则放弃追击。

**第一版不做寻路**：MoveTowards 直线追击。敌人会被墙卡住，但这在第一版是可接受的。后期可以接入 A* 或 NavMesh。

#### Attack 状态

```
UpdateAttack():
    if (targetPlayer == null):
        切换到 Idle
        return
    
    float distance = Vector2.Distance(position, targetPlayer.position)
    
    // 如果目标离开攻击范围，回到追击
    if (distance > attackRange * 1.2):  // 1.2 倍缓冲，防止反复切换
        切换到 Chase
        return
    
    // 停止移动
    rb.velocity = Vector2.zero
    
    // 朝向玩家
    Vector2 direction = (targetPlayer.position - position).normalized
    RotateTowards(direction)
    
    // 攻击冷却检查
    if (Time.time >= lastAttackTime + attackInterval):
        StartCoroutine(AttackCoroutine())
```

**攻击协程：**
```
AttackCoroutine():
    // 攻击前摇（视觉提示：可以改变颜色/缩放）
    spriteRenderer.color = Color.yellow  // 临时：攻击预警
    yield return WaitForSeconds(attackWindup)
    
    // 执行攻击：检测攻击范围内的玩家
    Collider2D hit = Physics2D.OverlapCircle(position, attackRange, detectionLayer)
    if (hit != null):
        Damageable playerDamageable = hit.GetComponent<Damageable>()
        if (playerDamageable != null):
            playerDamageable.TakeDamage(attackDamage, gameObject)
    
    lastAttackTime = Time.time
    spriteRenderer.color = chaseColor  // 恢复颜色
```

#### Dead 状态

```
OnDeath():  // 由 Damageable.OnDeath 事件触发
    currentState = Dead
    rb.velocity = Vector2.zero
    
    // 禁用碰撞
    GetComponent<Collider2D>().enabled = false
    
    // 播放死亡效果
    StartCoroutine(DeathCoroutine())

DeathCoroutine():
    // 死亡视觉效果：缩小 + 淡出
    float duration = 0.5
    float t = 0
    Vector3 originalScale = transform.localScale
    Color originalColor = spriteRenderer.color
    
    while (t < duration):
        t += Time.deltaTime
        float ratio = t / duration
        transform.localScale = Vector3.Lerp(originalScale, Vector3.zero, ratio)
        spriteRenderer.color = new Color(originalColor.r, originalColor.g, originalColor.b, 1 - ratio)
    
    // 销毁自身
    Destroy(gameObject)
```

### 2.6 辅助方法

```
RotateTowards(Vector2 direction):
    float targetAngle = Mathf.Atan2(direction.y, direction.x) * Mathf.Rad2Deg
    float currentAngle = rb.rotation
    float newAngle = Mathf.MoveTowardsAngle(currentAngle, targetAngle, rotationSpeed * Time.deltaTime)
    rb.MoveRotation(newAngle)

// 状态切换（带日志，方便调试）
ChangeState(EnemyState newState):
    if (currentState == newState) return
    if (currentState == Dead) return  // 死亡后不能切换状态
    
    // 退出旧状态
    OnExitState(currentState)
    
    currentState = newState
    
    // 进入新状态
    OnEnterState(newState)

OnEnterState(state):
    switch(state):
        Idle:   spriteRenderer.color = Color.white   // 白色 = 待机
        Chase:  spriteRenderer.color = Color.red     // 红色 = 追击
        Attack: spriteRenderer.color = Color.red     // 红色 = 攻击中

OnExitState(state):
    switch(state):
        Chase:  rb.velocity = Vector2.zero  // 停止移动
        Attack: // 清理攻击状态
```

## 四、与现有系统的对接

### 4.1 Layer 系统

敌人 GameObject 设置为 `Enemy` 层。现有的 `BulletBase.SetBulletLayer()` 逻辑：
- 玩家射出的子弹 → `FriendlyFire` 层
- `FriendlyFire` 的 hitLayerMask 排除了 `Player` 和 `Ally` 层
- 因此玩家子弹**可以命中 Enemy 层**的对象 ✅

**不需要修改任何 Layer 逻辑。**

### 4.2 GameSceneManager

敌人需要从 `GameSceneManager.Instance.CurrentPlayerCharacter` 获取玩家引用。

```csharp
// 在 EnemyBase.Start() 中
var playerCharacter = GameSceneManager.Instance?.CurrentPlayerCharacter;
if (playerCharacter != null)
{
    playerTransform = playerCharacter.transform;
}
```

### 4.3 Damageable 事件

敌人的 `EnemyBase` 在 Awake 中订阅同 GameObject 上 `Damageable` 的 `OnDeath` 事件，用于触发死亡状态。

### 4.4 受击反馈

敌人 Prefab 上挂载 `DamageFlash`（任务 1），被子弹命中时自动闪白。
可选：挂载 `DamageAudio` 播放受击音效。

### 5.1 Gizmos 绘制

在 `EnemyBase` 中实现 `OnDrawGizmosSelected()`：

```
OnDrawGizmosSelected():
    // 检测范围（绿色）
    Gizmos.color = Color.green
    Gizmos.DrawWireSphere(position, detectionRadius)
    
    // 丢失目标范围（黄色）
    Gizmos.color = Color.yellow
    Gizmos.DrawWireSphere(position, loseTargetRadius)
    
    // 攻击范围（红色）
    Gizmos.color = Color.red
    Gizmos.DrawWireSphere(position, attackRange)
```

### 5.2 状态颜色指示

利用 `SpriteRenderer.color` 直观显示当前状态：
- **白色**：Idle（待机）
- **红色**：Chase（追击中）
- **黄色**：Attack 前摇（即将攻击）
- **灰色 + 缩小**：Dead（死亡中）

## 八、注意事项

1. **不要做寻路**：第一版 MoveTowards 直线追击，被墙卡住是可接受的
2. **不要做巡逻**：Idle 就是站桩，巡逻路径是后期功能
3. **不要做视野锥**：OverlapCircle 全向检测，视野锥是后期优化
4. **不要做多种敌人**：先做一个近战敌人，验证闭环后再扩展
5. **攻击前摇很重要**：0.3 秒的前摇给玩家视觉预警，这是公平性设计的基础
6. **Freeze Rotation**：Rigidbody2D 锁定旋转，防止物理碰撞导致敌人旋转抖动，旋转由代码控制


# 阶段三：搭建第一个完整关卡 / Demo

> 预计工期：2~4 周
> 前置依赖：阶段一（可玩性闭环）+ 阶段二（关卡基础设施）完成
> 核心目标：设计并搭建一个可以从头玩到尾的完整关卡，验证所有系统的协同工作

---

## 为什么这是最终目标？

前两个阶段搭建的是"引擎"和"工具"，这个阶段才是真正的"造车"。
一个完整的可玩 Demo 是：
- 向自己证明：这些系统能协同工作
- 向玩家/发行商展示：这个游戏值得期待
- 发现问题：只有完整流程才能暴露系统间的缝隙

---

## 工作流程

### 第一步：关卡设计文档（纸面设计）

在动手搭建之前，先在文档中完成以下设计：

#### 1.1 关卡概述

- 关卡名称 / 代号
- 关卡背景故事（一段话即可）
- 关卡目标（玩家需要做什么才能通关）
- 预计游玩时长

#### 1.2 关卡流程设计

用流程图描述玩家的体验路径：

```
开场演出 → 探索区域 A → 战斗遭遇 1 → 解谜/交互 → 
过渡演出 → 探索区域 B → 战斗遭遇 2 → Boss 战 → 
结局演出 → 关卡结束
```

#### 1.3 节奏曲线

设计关卡的紧张度曲线：

```
紧张度
  ↑
  │      ╱╲        ╱╲╱╲
  │    ╱    ╲    ╱        ╲
  │  ╱        ╲╱            ╲
  │╱                          ╲
  └──────────────────────────────→ 时间
  开场  探索  战斗1  喘息  Boss  结局
```

关键原则：
- **张弛有度**：战斗之间要有探索/叙事的喘息空间
- **逐步升级**：后面的战斗比前面的更激烈
- **高潮收尾**：Boss 战或关键事件放在最后

#### 1.4 敌人配置表

| 遭遇编号 | 位置 | 敌人类型 | 数量 | 触发条件 | 特殊规则 |
|----------|------|----------|------|----------|----------|
| 1 | 区域 A 走廊 | 近战小兵 | 3 | 进入 TriggerZone | 锁门，全灭后开门 |
| 2 | 区域 B 广场 | 近战小兵 + 远程 | 2+1 | 进入 TriggerZone | 分波刷出 |
| Boss | 区域 B 尽头 | Boss | 1 | 进入 TriggerZone | 镜头演出后开始 |

#### 1.5 剧情 / 对话节点

| 节点编号 | 位置 | 触发条件 | 对话内容概要 | 后续事件 |
|----------|------|----------|-------------|----------|
| D1 | 开场 | 场景加载后 | 主角独白/任务说明 | 无 |
| D2 | 区域 A NPC | 接近 NPC | NPC 提供线索 | 标记目标位置 |
| D3 | Boss 前 | 进入 Boss 区域 | Boss 台词 | 开始 Boss 战 |

#### 1.6 物品 / 资源分布

| 物品 | 位置 | 获取方式 | 说明 |
|------|------|----------|------|
| 左轮手枪 | 区域 A 桌上 | 拾取 WorldItem | 玩家的第一把武器 |
| 弹药补给 | 区域 A→B 过渡 | 拾取 | 战斗前补给 |
| 回复道具 | 区域 B 角落 | 拾取 | Boss 战前补给 |

---

### 第二步：Greybox 搭建（灰盒原型）

**目标**：用简单的方块和基础形状搭出关卡的空间布局，验证尺度和节奏。

#### 2.1 搭建原则

- **不用美术资产**：全部用 Unity 默认的 Sprite（方块/圆形）+ 颜色区分
- **标注功能区域**：用不同颜色标记（红色 = 战斗区，蓝色 = 安全区，绿色 = 交互点）
- **放置 TriggerZone**：在关键位置放置触发区域，配置 UnityEvent
- **放置敌人生成点**：用空 GameObject + Gizmo 标记
- **放置物品**：用 WorldItem 预制体

#### 2.2 验证清单

- [ ] 玩家从出生点到终点的路径是否清晰？
- [ ] 各区域的大小是否合适？（太大会无聊，太小会拥挤）
- [ ] 战斗区域是否有足够的掩体和走位空间？
- [ ] 物品放置位置是否合理？（玩家能自然发现）
- [ ] 节奏是否符合设计？（走一段打一段，不会连续高强度）

#### 2.3 试玩迭代

- 自己试玩 3~5 遍
- 记录不舒服的地方
- 调整空间布局和敌人配置
- 重复直到手感满意

---

### 第三步：Gameplay 脚本编写

**目标**：用阶段二的事件系统 + 一次性脚本，实现关卡的完整流程。

#### 3.1 开场演出脚本

```
场景加载 → 镜头从远处推近到玩家 → 播放开场对话 → 恢复玩家控制
```

#### 3.2 战斗遭遇脚本

```
玩家进入 TriggerZone → 锁门 → 刷出敌人 → 监听敌人全灭 → 开门 → 触发下一段
```

#### 3.3 Boss 战脚本

```
玩家进入 Boss 区域 → 镜头演出（展示 Boss）→ Boss 台词 → 
BGM 切换 → Boss 战开始 → Boss 死亡 → 结局演出
```

#### 3.4 脚本放置

- 放在 `Assets/Scenes/{场景名}/场景代码/` 目录下
- 命名格式：`Level{场景名}_{功能描述}.cs`

---

### 第四步：美术替换

**目标**：用正式美术资产替换 Greybox。

#### 4.1 替换顺序

1. **地面 Tilemap**：用已有的城市地砖 Tile Palette
2. **墙体**：用已有的墙壁预制体（墙壁1/墙壁2）
3. **场景装饰**：灯、遮阳伞、空调外机、防盗网等已有预制体
4. **前景物体**：挂载 ForegroundObject，设置虚拟高度
5. **光照**：2D Light 布置（已有 URP 2D Lit 管线）
6. **角色/敌人 Sprite**：替换临时 Sprite 为正式像素画

#### 4.2 注意事项

- 替换美术时**不要改变碰撞体**，保持 Greybox 阶段验证过的空间尺度
- 前景物体记得开启纹理的 **Read/Write Enabled**（ForegroundManager 模糊需要）
- 光照布置参考已有的 Scene0 咖啡厅灯效果

---

### 第五步：音频铺设

**目标**：为关卡铺设完整的音频体验。

#### 5.1 音频清单

| 类型 | 内容 | 实现方式 |
|------|------|----------|
| BGM | 探索 BGM / 战斗 BGM / Boss BGM | MusicManager 切换 |
| 环境音 | 城市环境 / 室内环境 | FMOD Snapshot Trigger |
| 脚步声 | 不同地面材质 | 已有 FMOD_FootStepPlayer_Parameter_Trigger |
| 战斗音效 | 枪声/受击/死亡 | 已有射击音效 + 新增受击/死亡音效 |
| 交互音效 | 开门/拾取/对话推进 | FMOD 一次性事件 |
| 演出音效 | 镜头移动/Boss 登场 | 关卡脚本中触发 |

#### 5.2 铺设步骤

1. 放置 FMOD_FootStepPlayer_Parameter_Trigger（地面材质区域）
2. 放置 FMOD_Snapshot_Trigger（环境音区域）
3. 在关卡脚本中配置 BGM 切换点
4. 为新增的交互/战斗音效创建 FMOD 事件

---

### 第六步：打磨

**目标**：反复测试，打磨细节。

#### 6.1 打磨清单

- [ ] 战斗手感：射击反馈、敌人受击反馈、死亡特效
- [ ] 移动手感：速度、加速度、转向是否舒适
- [ ] 镜头：跟随弹性、演出流畅度、抖动强度
- [ ] 音频：音量平衡、空间感、遮挡效果
- [ ] 节奏：战斗密度、探索时长、演出节奏
- [ ] 难度：敌人数量、伤害数值、物品分布
- [ ] Bug：碰撞穿模、事件未触发、音频异常

#### 6.2 迭代方法

```
试玩 → 记录问题 → 分类（手感/节奏/Bug）→ 修复 → 再试玩
```

重复 3~5 轮，直到自己觉得"这个关卡我愿意让别人玩"。

---

## 验收标准

完成阶段三后，应该有一个**完整的可玩 Demo**：

- ✅ 玩家可以从开场玩到结局，体验完整的关卡流程
- ✅ 有至少 2~3 次战斗遭遇 + 1 次 Boss 战（或等价的高潮事件）
- ✅ 有对话/叙事推进剧情
- ✅ 有镜头演出增强沉浸感
- ✅ 有完整的音频体验（BGM + 环境音 + 音效）
- ✅ 美术风格统一，视觉效果完整
- ✅ 无严重 Bug，流程可以顺畅走完

---

## 建议的开发顺序

```
Week 1:   关卡设计文档 + Greybox 搭建 + 试玩迭代
Week 2:   Gameplay 脚本编写 + 流程串联
Week 3:   美术替换 + 音频铺设
Week 4:   打磨 + Bug 修复 + 最终测试
```

---

## 注意事项

1. **Greybox 阶段是最重要的**：不要急着上美术。灰盒阶段验证的空间尺度和节奏，决定了关卡的最终质量。
2. **先做最小可玩版本**：第一遍不需要完美，先把流程跑通，再逐步打磨。
3. **记录设计决策**：为什么这里放 3 个敌人而不是 5 个？为什么这里有一段对话？把理由写下来，方便后续迭代时回顾。
4. **找人试玩**：自己测试容易"灯下黑"，找朋友试玩能发现很多自己注意不到的问题。
5. **控制范围**：第一个 Demo 不要太长，5~10 分钟的游玩时长就够了。做精比做多重要。

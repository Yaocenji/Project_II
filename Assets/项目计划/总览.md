
# Project_II 开发计划总览

> 最后更新：2026-04-16
> 当前阶段：**阶段一 · 补齐可玩性闭环**

---

## 一、项目定位

Unity 2D 俯视角（90° 纯俯视）像素风独立游戏。
技术栈：Unity URP 2D Lit + FMOD 音频 + Cinemachine 摄像机 + 新输入系统（Input System）。

---

## 二、当前系统完成度

### ✅ 已完成

| 分类 | 系统 | 备注 |
|------|------|------|
| **框架层** | GameSceneManager | 场景单例，注册/注销玩家角色、鼠标、可交互物品 |
| | InputManager | 统一输入管理，设备类型检测（键鼠/手柄） |
| | FXManager | 特效管理器（目前为空壳，待充实） |
| | 代码规范 | 命名空间、命名规范、Git 提交规范、.cursorrules |
| **角色控制** | CharacterController | 移动/旋转/速度状态（静止/慢走/行走/奔跑） |
| | Mouse | 虚拟鼠标（键鼠 + 手柄双模式） |
| | FootstepAudio | 脚步声系统（距离累加触发、地面材质参数） |
| | FMOD_FootStepPlayer_Parameter_Trigger | 地面材质触发器盒 |
| **物品系统** | Item.Base | 物品基类（7 个交互虚方法 + OnEquip/OnUnequip） |
| | Hotbar | 快捷栏（输入转发、物品管理、跨容器转移、丢弃） |
| | Backpack | 背包（堆叠、自动放入、存取） |
| | WorldItem | 世界掉落物（拾取交互、实例化手持物） |
| **武器系统** | WeaponBase | 武器基类（继承 Item.Base，纯虚方法） |
| | BulletBase | 子弹基类（即时命中 / 物理实体、Layer 自动设置） |
| | Revolver | 左轮手枪（完整实现：射击/换弹/散布/对象池） |
| | BulletRevolver | 左轮子弹 |
| | ShootInstructionLine | 射击提示线 UI |
| | BulletRevolverFX_Tracer | 曳光弹特效 |
| **渲染/视觉** | ForegroundManager + ForegroundObject | 前景管理器（虚拟高度驱动 Scale/模糊/偏移） |
| | CinemachineShake | 镜头抖动（射击/受击等） |
| | URP 2D Lit | 含自研 Radiance Cascades Package |
| **场景交互** | InteractableBase | 可交互物品抽象基类 |
| | Door1 | 门（铰链物理、开/关交互） |
| **音频** | FMOD 集成 | 枪声/脚步声/环境音 |
| | 墙壁遮挡参数 | 射线检测计算遮挡距离 |
| | 地面材质参数切换 | 触发器驱动 |
| **美术资产** | demo 级素材 | 米画师一期 + 自绘（墙体/灯/空调外机/防盗网/门等） |
| | Tile Palette | 城市地砖 |
| | 场景预制体 | 墙壁/灯/遮阳伞/咖啡厅灯/门 |

### ❌ 缺失 / 薄弱

| 系统 | 重要程度 | 说明 |
|------|----------|------|
| 生命值 / 伤害系统 | 🔴 关键 | 子弹 Hit 后无伤害逻辑 |
| 敌人 / AI 系统 | 🔴 关键 | 无任何敌人实现 |
| 玩家死亡 / 重生 | 🔴 关键 | 无 HP 归零处理 |
| HUD（弹药/HP/装备显示） | 🟡 重要 | 无任何游戏内 UI |
| 关卡事件 / 脚本系统 | 🟡 重要 | 无触发式事件框架 |
| 对话 / 剧情系统 | 🟡 重要 | 无对话框和文本系统 |
| 场景切换 / 加载 | 🟡 重要 | 有多个场景但无切换逻辑 |
| BGM / 环境音管理 | 🟠 一般 | FMOD 素材已有，缺管理器 |
| 镜头演出工具 | 🟠 一般 | Cinemachine 已有，缺过场系统 |
| 存档 / 读档 | ⚪ 后期 | 可在 Demo 完成后再做 |
| 策划文档 / GDD | 🟡 重要 | 无游戏设计文档 |

## 三、开发路线图

```
当前位置：
  ✅ 底层框架 → ✅ 核心系统 → ⬅️【你在这里】→ 阶段一 → 阶段二 → 阶段三
```

### 🔴 阶段一：补齐可玩性闭环（预计 1~2 周）

> 目标：形成"移动→射击→敌人受伤→敌人反击→玩家受伤→死亡/胜利"的完整战斗循环。

详见 → [阶段一：可玩性闭环.md](阶段一：可玩性闭环.md)

### 🟡 阶段二：关卡基础设施与演出工具（预计 1~2 周）

> 目标：搭建关卡设计所需的工具链——事件系统、对话系统、场景切换、BGM 管理、镜头演出。

详见 → [阶段二：关卡基础设施.md](阶段二：关卡基础设施.md)

### 🟢 阶段三：搭建第一个完整关卡 / Demo（预计 2~4 周）

> 目标：设计并搭建一个可以从头玩到尾的完整关卡，验证所有系统的协同工作。

详见 → [阶段三：搭建第一个关卡.md](阶段三：搭建第一个关卡.md)

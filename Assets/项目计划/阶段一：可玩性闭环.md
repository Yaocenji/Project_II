
# 阶段一：补齐可玩性闭环

> 预计工期：1~2 周
> 前置依赖：无（当前系统已满足）
> 核心目标：形成完整的战斗循环，能够反复试玩并调试手感

---

## 为什么这是最优先的？

独立游戏的核心是 **gameplay feel**（游戏手感）。
目前项目的射击系统已经相当完善（散布、弹药、换弹、对象池、曳光弹、镜头抖动、音效遮挡），
但子弹命中后**什么都不会发生**——没有伤害、没有敌人、没有死亡。

没有"射击→造成伤害→敌人死亡"这个闭环，就无法验证：
- 射击手感是否舒服
- 伤害数值是否合理
- 战斗节奏是否紧凑
- 关卡设计的敌人配置是否有趣

**先有闭环，再搭关卡。**

## 任务清单

### 1. 生命值 / 伤害系统 🔴 P0

#### 1.1 Damageable 组件

**目标**：任何可以被伤害的对象（玩家、敌人、可破坏物）都挂载此组件。

**设计要点**：
- 创建 `ProjectII.Combat.Damageable` 组件（MonoBehaviour）
- 成员：
  - `maxHP`：最大生命值（int，Inspector 可调）
  - `currentHP`：当前生命值（int）
  - `isInvincible`：是否无敌（bool，用于受击无敌帧）
  - `invincibleDuration`：无敌帧持续时间（float）
- 公有方法：
  - `TakeDamage(int damage, GameObject source)`：受到伤害
    - 如果 `isInvincible` 为 true，忽略
    - 扣减 HP，触发 `OnDamaged` 事件
    - 如果 HP ≤ 0，触发 `OnDeath` 事件
  - `Heal(int amount)`：恢复生命值
- 事件（UnityEvent 或 C# event）：
  - `OnDamaged(int damage, GameObject source)`
  - `OnDeath()`
  - `OnHPChanged(int currentHP, int maxHP)`
- 受击无敌帧：`TakeDamage` 后启动协程，`invincibleDuration` 秒内 `isInvincible = true`

**放置位置**：`Assets/Combat/Damageable.cs`
**命名空间**：`ProjectII.Combat`

#### 1.2 子弹命中接入伤害

**目标**：子弹 Hit 时调用目标的 `Damageable.TakeDamage`。

**修改点**：
- `BulletBase` 或其子类的 `Hit` 方法中：
  - 获取被命中对象的 `Damageable` 组件
  - 如果存在，调用 `TakeDamage(damage, shooter)`
  - `damage` 值可以由子弹自身携带，或从武器数据中获取
- `BulletRevolver` 的 `Hit` 实现中接入

#### 1.3 受击反馈

**目标**：被击中时有明确的视觉/音频反馈。

**内容**：
- 受击闪白（SpriteRenderer 材质闪烁，或 shader 参数）
- 受击音效（FMOD 事件）
- 击退效果（可选，给 Rigidbody2D 施加冲量）
- 可以在 `Damageable.OnDamaged` 事件中挂载这些反馈

### 2. 最简敌人 🔴 P0

#### 2.1 敌人基类

**目标**：创建一个最简单的敌人，能被打死，能打玩家。

**设计要点**：
- 创建 `ProjectII.Enemy.EnemyBase`（MonoBehaviour）
- 挂载 `Damageable` 组件
- 最简 AI 状态机（枚举驱动）：
  ```
  Idle → Patrol → Chase → Attack → Dead
  ```
- 第一版只需实现：
  - **Idle**：站桩不动
  - **Chase**：发现玩家后直线追击（简单的 MoveTowards）
  - **Attack**：进入攻击范围后攻击（近战：直接调用玩家 Damageable.TakeDamage）
  - **Dead**：HP 归零后播放死亡动画/特效，销毁自身
- 发现玩家：
  - 简单方案：固定检测半径（OverlapCircle）
  - 进阶方案：视野锥 + 射线检测（可后期再做）

**放置位置**：`Assets/Enemy/EnemyBase.cs`
**命名空间**：`ProjectII.Enemy`

#### 2.2 第一个具体敌人

**目标**：一个近战敌人，用于测试战斗循环。

- 继承 `EnemyBase`
- 使用临时 Sprite（方块/圆形即可）
- 巡逻路径可选（第一版可以不做）
- 攻击方式：接触伤害或近战挥砍
- 攻击间隔、伤害值在 Inspector 中可调

---

### 3. 玩家死亡 / 重生 🔴 P0

**目标**：玩家 HP 归零后有明确的处理。

**设计要点**：
- 玩家角色挂载 `Damageable` 组件
- 监听 `OnDeath` 事件：
  - 第一版：直接重新加载当前场景（`SceneManager.LoadScene(SceneManager.GetActiveScene().name)`）
  - 后期：死亡动画 → 黑屏 → 重生点/检查点
- 监听 `OnDamaged` 事件：
  - 受击屏幕闪红（Post Processing Volume 或 UI 叠加层）
  - 受击镜头抖动（复用已有的 CinemachineShake）

---

### 4. 最简 HUD 🟡 P1

**目标**：在屏幕上显示关键游戏数据，不需要精美，能看到数据就行。

**内容**：
- HP 显示（数字或简易血条）
- 当前弹药数 / 最大弹药数
- 当前装备物品图标
- 快捷栏格子指示（当前选中高亮）

**设计要点**：
- 创建 `ProjectII.UI.GameHUD`（MonoBehaviour）
- 挂载在 Canvas 上
- 每帧（或事件驱动）从以下来源读取数据：
  - HP：玩家角色的 `Damageable` 组件
  - 弹药：当前装备武器的弹药数据（需要在 WeaponBase 或 Revolver 中暴露）
  - 装备：`Hotbar` 的当前装备格
- 使用 Unity 默认 UI 组件（Text/Image），后期再替换为美术 UI

**放置位置**：`Assets/UI/GameHUD.cs`
**命名空间**：`ProjectII.UI`

## 验收标准

完成阶段一后，应该能在游戏中实现以下完整流程：

```
玩家出生 → 移动探索 → 发现敌人 → 射击敌人 → 敌人受伤（有反馈）→ 敌人死亡
                                    ↓
                              敌人反击 → 玩家受伤（有反馈）→ HP 显示变化
                                    ↓
                              玩家 HP 归零 → 死亡 → 重新加载场景
```

---

## 建议的开发顺序

```
Day 1-2: Damageable 组件 + 子弹接入伤害 + 受击反馈
Day 3-4: 最简敌人（站桩靶子 → 追击 → 攻击）
Day 5:   玩家死亡/重生
Day 6-7: 最简 HUD + 整体联调
```

---

## 注意事项

1. **不要过早优化 AI**：第一版敌人只需要能跑能打就行，复杂的行为树/状态机留到后面。
2. **不要过早做美术**：敌人用临时 Sprite，HUD 用默认 UI，先验证 gameplay。
3. **建议在 Scene2 中搭建"战斗沙盒"**：放几个敌人靶子，反复调试射击手感和伤害数值。
4. **FXManager 可以在此阶段充实**：管理受击特效、死亡特效、屏幕闪红等。
5. **暴露武器数据给 HUD**：Revolver 的 `currentAmmo` 和 `maxAmmo` 目前是 private，需要添加公有属性或事件供 HUD 读取。
